Conceptual Architecture

High-level dataflow: user → router → retrieval → reranker → LLM synthesis, with corpus and indexes providing context, and evaluation feeding improvements.

flowchart TB User[User] --> Query[User query] Query --> Router[Query router: tag intent] Router --> Retrieval[Retriever: BM25 + hybrid + boosts] Retrieval --> Candidates[Top-N documents] Candidates --> Reranker[Cross-encoder reranker] Reranker --> TopK[Top-5 ranked] TopK --> LLM[LLM synthesis with citations] LLM --> Answer[Grounded answer] Corpus[Curated corpus] --> Indexes[Indexes: BM25, FAISS] Indexes --> Retrieval Answer --> Eval[Evaluation] Eval -. offline .-> Router Eval -. offline .-> Retrieval
